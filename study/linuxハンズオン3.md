# 🧩 Linux ハンズオン３

## ゴール
- sed / awk / xargs を使ってテキスト加工ができる
- ログや設定ファイルを自動で編集できる

---

1. 作業ディレクトリへ移動

```
# cd: ディレクトリ移動
# `~` はホームディレクトリ
cd ~/linux_cicd_training/workspace

# pwd: 現在のディレクトリ（作業場所）を表示
pwd
```

2. 特定文字列の置換（ユースケース：ログの簡易整形）

```
# sed: 1行ずつ読み、指定ルールで置換/削除/抽出できる
# s/AAA/BBB/: AAA を BBB に置換
# g: 行内の全てを置換（global）
sed 's/ERROR/ERR/g' logs/app.log
```

3. ファイルを直接書き換える（ユースケース：ログ修正）

```
# -i: ファイルを直接書き換える（in-place）
# 注意: WSL(GNU sed) 前提（macOSのsedはオプションが異なる）
sed -i 's/WARN/WARNING/g' logs/app.log
```

4. 特定行だけ表示（ユースケース：ログ抽出）

```
# -n: デフォルト出力を止める（必要な行だけ出すため）
# '1,5p': 1〜5行目を表示（p = print）
sed -n '1,5p' logs/app.log
```

5. 特定行削除（ユースケース：不要ログ削除）

```
# '1,2d': 1〜2行目を削除して出力（d = delete）
# ※ファイル自体は書き換えない（-i を付けない限り）
sed '1,2d' logs/app.log
```

6. 条件付き抽出（ユースケース：ERROR抽出）

```
# '/ERROR/p': ERROR にマッチした行だけ表示
sed -n '/ERROR/p' logs/app.log
```

7. カラム抽出（ユースケース：ログ分析）

```
# awk: 「列（フィールド）」を扱いやすいテキスト処理
# $1, $2 ... で列を指定する（スペース区切りが基本）
awk '{print $1, $2}' logs/app.log
```

8. 条件付き抽出（ユースケース：ERRORだけ）

```
# /ERROR/ ...: ERRORにマッチする行だけ処理
# $0: 行全体
awk '/ERROR/ {print $0}' logs/app.log
```

9. 特定カラム条件（ユースケース：ユーザー抽出）

```
# $4: 4列目（例: user=tanaka）
# ~ /tanaka/: 正規表現でマッチするか判定
awk '$4 ~ /tanaka/' logs/app.log
```

10. 件数カウント（ユースケース：集計）

```
# count++: 行ごとにcountを増やす
# END { ... }: 最後に1回だけ実行
awk '{count++} END {print count}' logs/app.log
```

11. ERROR件数のみカウント（ユースケース：障害分析）

```
# ERROR行だけ count++ して、最後に表示
awk '/ERROR/ {count++} END {print count}' logs/app.log
```

12. CSVの列抽出（ユースケース：ユーザーデータ確認）

```
# -F',': 区切り文字をカンマに指定（CSV用）
# $2: 2列目（name）
awk -F',' '{print $2}' search/users.csv
```

13. 条件付きCSV抽出（ユースケース：activeユーザー）

```
# $4=="active": 4列目がactiveの行だけ出す
awk -F',' '$4=="active" {print $2}' search/users.csv
```

14. 複数コマンド連携（ユースケース：エラー分析）

```
# grep: 文字列検索
# | (パイプ): 左の出力を右の入力へ渡す
# ERROR行から user=... を抜き出す
grep ERROR logs/app.log | awk '{print $4}'
```

15. ファイル一覧取得（ユースケース：一括処理）

```
# ワイルドカードで複数ファイルの一覧を出す
ls logs/*.log
```

16. xargsで一括処理（ユースケース：複数ファイル検索）

```
# xargs: 標準入力から受け取った文字列を「引数」としてコマンド実行する
# ここでは、ログファイル一覧を grep の引数に渡す
ls logs/*.log | xargs grep ERROR
```

17. 削除処理（ユースケース：一括削除）

```
# 注意: rm は危険。まず対象を ls で確認する
ls tmp/*.txt

# rm -i: 削除前に確認（事故防止）
ls tmp/*.txt | xargs rm -i
```

18. 安全な削除（ユースケース：確認付き削除）

```
# xargs -p: 実行前に毎回確認（prompt）
ls tmp/*.txt | xargs -p rm
```

19. 空白対応（ユースケース：安全処理）

```
# find -print0: ファイル名の区切りを \0（NUL）にする（空白・改行に強い）
# xargs -0: \0 区切りとして受け取る
find . -name "*.log" -print0 | xargs -0 grep ERROR
```

20. コマンド組み合わせ（ユースケース：実務ログ解析）

```
# user=... を集計して多い順に並べる
# sort | uniq -c | sort -nr が定番
grep ERROR logs/app.log | awk '{print $4}' | sort | uniq -c | sort -nr
```

21. sed + awk（ユースケース：整形＋抽出）

```
# ERROR を ERR に置換してから、日付とユーザー列だけ表示
sed 's/ERROR/ERR/g' logs/app.log | awk '{print $1, $4}'
```

22. 複数条件処理（ユースケース：高度分析）

```
# awk 内で AND 条件
awk '/ERROR/ && /tanaka/ {print $0}' logs/app.log
```

23. ログからCSV作成（ユースケース：データ変換）

```
# カンマ区切りで出力してCSVっぽくする
awk '{print $1","$2","$4}' logs/app.log
```

24. CI/CD前処理（ユースケース：設定変更）

```
# sed -i で設定を書き換える（例: dev -> prod）
sed -i 's/ENV=dev/ENV=prod/' src/config/app.conf
```

25. CI/CD後処理（ユースケース：ログ要約）

```
# >: リダイレクト（コマンド出力をファイルに保存）
# ERROR行から日付とユーザーを抜き出して保存
grep ERROR logs/app.log | awk '{print $1, $4}' > error_summary.txt
```

## 学び

① sed：文字列加工

```
# 置換: s/AAA/BBB/g
# 抽出: -n '1,5p' / -n '/PAT/p'
# 削除: '1,2d'
```

② awk：構造データ処理

```
# 列: $1 $2 ...
# 区切り: -F','
# 条件: /ERROR/ や $4=="active"
# 集計: END {print count}
```

③ xargs：一括処理

```
# 標準入力 → 引数化して実行
# 安全: -p
# 空白対応: find -print0 | xargs -0
```

## おまけ：vimの使い方（最小）

```
# vim: エディタ
# --- モード ---
# ノーマル（=コマンド）モード: 起動直後の状態（基本ここ）
    #   - 目的: 文字入力ではなく「操作」をする（移動/削除/コピー/貼り付け/検索 など）
#   - 例: gg, G, dd, yy, p, /ERROR, u
# インサートモード: 文字を入力する
# ビジュアルモード: 範囲選択する
# コマンドラインモード: 画面下に : が出て、1行の命令を入力する
#   - 目的: ファイル操作や設定、置換など「コマンド」を1行で実行する
#   - 例: :w, :q, :set number, :%s/ERROR/ERR/g
#
# 違い（ここが重要）:
#   - ノーマルモード = 画面上のカーソル位置に対して操作する（キー1〜数文字の操作が多い）
#   - コマンドラインモード = 画面下の1行に「: で始まる命令」を入力して実行する
#
# 戻り方:
#   - コマンドラインモードは Enter で実行、Esc でキャンセルしてノーマルへ戻る
#
# Esc: どのモードからでもノーマルモードへ戻る
# i（ノーマル）: インサートモードに入る
# v（ノーマル）: ビジュアルモードに入る
# V（ノーマル）: 行単位のビジュアルモード
# :（ノーマル）: コマンドラインモードに入る
#
# :w（コマンドライン）: 保存
# :q（コマンドライン）: 終了
# :wq（コマンドライン）: 保存して終了
# :q!（コマンドライン）: 保存せず終了

# --- 移動 ---
# h j k l（ノーマル）: 左 下 上 右（基本移動）
# gg（ノーマル）: ファイル先頭へ
# G（ノーマル）: ファイル末尾へ
# 10G（ノーマル）: 10行目へ
# 0（ノーマル）: 行頭へ
# $（ノーマル）: 行末へ

# --- 検索 ---
# /word（ノーマル）: 下方向に検索
# ?word（ノーマル）: 上方向に検索
# n（ノーマル）: 次の一致
# N（ノーマル）: 前の一致

# --- コピー/貼り付け/削除（コマンドモード） ---
# yy（ノーマル）: 1行コピー
# 5yy（ノーマル）: 5行コピー
# p（ノーマル）: 下に貼り付け
# P（ノーマル）: 上に貼り付け
# dd（ノーマル）: 1行削除（=切り取り）
# 5dd（ノーマル）: 5行削除

# --- 範囲選択（ビジュアルモード） ---
# v（ノーマル）→ 移動キー: 文字単位で選択
# V（ノーマル）→ j/k: 行単位で選択
# y（ビジュアル）: 選択範囲をコピー
# d（ビジュアル）: 選択範囲を削除

# --- 編集 ---
# a（ノーマル）: カーソルの後ろから挿入（インサートへ）
# o（ノーマル）: 次の行を作って挿入（インサートへ）
# O（ノーマル）: 前の行を作って挿入（インサートへ）

# --- 取り消し/やり直し ---
# u（ノーマル）: 取り消し（Undo）
# Ctrl+r（ノーマル）: やり直し（Redo）

# --- 表示 ---
# :set number（コマンドライン）: 行番号表示ON
# :set nonumber（コマンドライン）: 行番号表示OFF
# :set paste（コマンドライン）: ペースト時の崩れ防止（必要なときだけ）
# :set nopaste（コマンドライン）: paste解除

# --- 置換（sedの s///g と同じ発想） ---
# :%s/ERROR/ERR/g（コマンドライン）: ファイル全体で置換
# :%s/ERROR/ERR/gc（コマンドライン）: 1件ずつ確認しながら置換

# 例）ファイルを開く
vim error_summary.txt
```
